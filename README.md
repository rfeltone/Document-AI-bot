# AI Document Analysis Bot 🤖📄

Этот проект представляет собой комплексное решение на базе искусственного интеллекта, состоящее из двух основных частей:
1. **Telegram-ui** — фронтенд-часть, реализованная в виде Telegram-бота. Она отвечает за взаимодействие с пользователями, прием файлов и вывод ответов.
2. **DocumentBot** — бэкенд-часть на Python. Занимается обработкой полученных документов, извлечением текста и взаимодействием с LLM для генерации интеллектуальных ответов.

## 🏗️ Архитектура проекта

Проект организован в виде монорепозитория (monorepo), который включает в себя и интерфейс (Telegram-бота), и бэкенд-логику.

```text
.
├── DocumentBot/       # Бэкенд на Python (обработка документов + интеграция с LLM)
└── Telegram-ui/       # Фронтенд на TypeScript (пользовательский интерфейс Telegram-бота)
```

### 🔹 Telegram-ui (Фронтенд)
- **Язык/Технологии**: TypeScript / Node.js / GrammY + Conversation
- **Назначение**: Принимает команды и документы от пользователя в Telegram, передает их на бэкенд и возвращает итоговый ответ.

### 🔹 DocumentBot (Бэкенд)
- **Язык/Технологии**: Python
- **Назначение**: Получает файлы от фронтенда, парсит их содержимое (например, PDF, DOCX), формулирует запрос к языковой модели (LLM) и возвращает готовый ответ.

## 🚀 Как запустить локально

Для запуска проекта вам понадобится настроить обе его части.

### Требования
- [Node.js](https://nodejs.org/) (для Telegram-ui)
- [Python 3.10+](https://www.python.org/) (для DocumentBot)

### 1️⃣ Настройка Telegram-ui
1. Перейдите в папку `Telegram-ui`:
   ```bash
   cd Telegram-ui
   ```
2. Установите зависимости:
   ```bash
   npm install
   ```
3. Создайте файл `.env` на основе шаблона и добавьте туда свой токен от бота:
   ```bash
   cp .env.example .env
   ```
4. Запустите проект:
   ```bash
   npm run dev
   ```

### 2️⃣ Настройка DocumentBot
1. Перейдите в папку `DocumentBot`:
   ```bash
   cd DocumentBot
   ```
2. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv .venv
   # Для Windows:
   .venv\Scripts\activate

   ```
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Запустите бэкенд:
   ```bash
   run.bat
   ```
